# מדיניות עיצוב Hoogi UI Sandbox

## סקירה כללית

מסמך זה מגדיר את מדיניות העיצוב והשימוש ב-Hoogi UI Sandbox. זהו מדריך מקיף למפתחים ומעצבים העובדים עם מערכת העיצוב של Hoogi.

## עקרונות יסוד

### 1. עיצוב בלבד
- **אין JavaScript**: כל הרכיבים והמסכים הם סטטיים בלבד
- **אין Database**: אין חיבור לנתונים או API
- **אין Framework**: רק HTML + Tailwind CSS דרך CDN
- **RTL מלא**: תמיכה מלאה בעברית וכיוון מימין לשמאל

### 2. עקביות עיצובית
- **טוקנים בלבד**: אסור להשתמש בצבעי hex קשיחים
- **משתני CSS**: כל הצבעים, ריווחים וגדלים מוגדרים ב-`_tokens.css`
- **רספונסיביות**: תמיכה מלאה ב-sm/md/lg עם Grid/Flex
- **נגישות**: תמיכה ב-focus states ו-keyboard navigation

### 3. מבנה קבצים
```
ui_sandbox/
├── design-tokens/
│   └── _tokens.css          # משתני עיצוב
├── components/
│   ├── button.html          # רכיב כפתורים
│   ├── input.html           # רכיבי קלט
│   ├── select.html          # רכיבי בחירה
│   ├── badge.html           # רכיב תגיות
│   ├── card.html            # רכיב כרטיסים
│   └── datatable.html       # רכיב טבלאות
├── screens/
│   ├── dashboard.html       # מסך לוח בקרה
│   ├── leads.html           # מסך לידים
│   ├── questionnaires.html  # מסך שאלונים
│   └── responses.html       # מסך תגובות
├── docs/
│   ├── policy.md            # מדיניות (זה)
│   ├── components-map.md    # מפת רכיבים
│   ├── screens-spec.md      # מפרטי מסכים
│   └── usage-guide.md       # מדריך שימוש
└── index.html               # דף ניווט ראשי
```

## כללי שימוש

### טוקנים
- **צבעים**: השתמש רק במשתני CSS המוגדרים ב-`_tokens.css`
- **ריווחים**: השתמש במשתני `--space-*` או Tailwind classes
- **טיפוגרפיה**: השתמש במשתני `--font-size-*` ו-`--font-weight-*`
- **רדיוסים**: השתמש במשתני `--radius-*`

### רכיבים
- **מצבים**: כל רכיב חייב לתמוך ב-hover, focus, disabled, error, loading
- **גדלים**: כל רכיב חייב לתמוך ב-sm, default, lg
- **וריאציות**: כל רכיב חייב לתמוך ב-primary, secondary, outline, ghost

### רספונסיביות
- **Mobile First**: עיצוב מתחיל ממובייל
- **Breakpoints**: sm (640px), md (768px), lg (1024px)
- **Grid System**: השתמש ב-CSS Grid ו-Flexbox
- **Overflow**: טבלאות עם overflow-x במובייל

## נקודות שבירה

### אסור בשום פנים:
1. **שינוי טוקנים**: אל תשנה את `_tokens.css` ללא אישור
2. **צבעים קשיחים**: אל תשתמש ב-hex colors ישירות
3. **JavaScript**: אל תוסיף JavaScript לרכיבים
4. **Framework**: אל תשתמש ב-React/Vue/Angular
5. **Database**: אל תנסה לחבר לנתונים

### מותר:
1. **העתקת HTML**: העתק את ה-HTML לרכיבים שלך
2. **שינוי תוכן**: שנה טקסטים ותמונות
3. **הוספת רכיבים**: צור רכיבים חדשים על בסיס הקיימים
4. **אינטגרציה**: השתמש עם כל Framework
